# Memory2307 — NFC 班级纪念网页设计文档

## 概述

为 2307 班打造一个通过 NFC 标签打开的纪念网页。每张 NFC 标签存储一个唯一 URL（如 `/#/s/a8k3p2`），对应一位学生。页面以全动画形式呈现班级回忆、教师寄语和录取通知书，风格为高级、简约、温暖的电影感。

## 技术栈

- **构建**: Vite + React + TypeScript
- **样式**: Tailwind CSS
- **动画**: Framer Motion
- **轮播**: MVP 使用 Framer Motion 自定义 crossfade（无需第三方轮播库）；后续如需手势滑动再引入 Embla Carousel
- **路由**: react-router-dom v6 + HashRouter
- **部署**: Cloudflare Pages（纯静态）
- **字体**: MVP 用系统字体（fallback 栈：`"PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`），后续可加思源宋体等 web font

## 路由方案

使用 HashRouter。URL 格式为 `/#/s/:slug`。NFC 标签写入此完整 hash URL。

路由表：
- `/#/s/:slug` → StudentPage（找到学生则渲染）
- `/#/s/:slug` → ErrorPage（未找到学生则渲染）
- `/` → 可选首页（MVP 阶段可跳过或做简单欢迎页）

## 页面流程

```
用户手机 NFC 碰标签
    ↓
打开 /#/s/a8k3p2
    ↓
Vite 静态 import students.json → 查找 slug
    ↓
找到 → StudentPage 渲染（三段动画自动播放）
未找到 → ErrorPage 渲染
```

## 动画编排

全局时间线（全自动播放，无需用户交互）：

```
T+0.0s   纯黑屏幕过渡（0.5s）
T+0.5s   ─── 第一段：照片轮播 ───
         10+ 张照片 crossfade + Ken Burns，每张 3-4s
         总时长约 8-10s
T+10s    轮播整体淡出（opacity: 0, scale: 1.02）
T+10.5s  ─── 第二段：教师寄语 ───
         页面淡入 + 打字机效果逐字出现
         总时长约 6-8s
T+18s    寄语整体淡出
T+18.5s  ─── 第三段：录取通知书展开 ───
         信封 → 封口打开 → 通知书滑出 → 内容浮现
         总时长约 8-10s
T+26s    通知书画面上移
T+27s    ─── 结尾定格 ───
         "敢赴山海，追梦星辰" 淡入
         "2307，未来可期" 延迟淡入
T+30s    停留，不再变化
```

## 三段动画详细设计

### 第一段：照片轮播

- 全屏展示，照片占满视口高度
- 照片交叉淡入淡出（crossfade）
- Ken Burns 效果：每张停留期间 `scale(1.0) → scale(1.05)`，配极微弱 `translateX/Y` 偏移
- 奇数张向左上推，偶数张向右下推，交替产生不重复运动感
- 底部叠加极淡渐变遮罩（透明→奶茶色），柔化过渡
- 使用 Framer Motion `AnimatePresence` 实现 crossfade，幻灯片用 `position: absolute` 叠放（MVP 不依赖 Embla）
- 结束时整体 `opacity: 0 + scale(1.02)` 淡出

### 第二段：教师寄语

- 背景：奶茶色 `#F5F0EB`
- 画面中央偏上，金色引号 `"` 装饰符号作为开场
- 教师签名在右上角，细衬线字体
- 逐字打字机效果，每字间隔 80ms
- 每字出现时带 `translateY(4px → 0)` 下落动画
- 光标闪烁：文字末尾 `|` 竖线，`opacity` 循环
- 文字出现时轻微粗细变化模拟"墨迹"感

### 第三段：录取通知书展开（信封拆信方案）

```
T+0.0s   信封居中淡入（scale 0.9 → 1，opacity 0 → 1）
T+0.8s   信封轻微上下浮动（breathing 动画）
T+1.5s   封口三角形向上翻开（rotateX 0 → -180°）
T+2.5s   通知书纸张从信封内上滑（translateY 100% → -20%）
T+4.0s   信封淡出下沉，通知书中枢化放大至 ~85% 屏宽
T+4.5s   通知书中内容逐行 staggered 淡入（每行间隔 0.3s）
T+7.0s   内容全部显示完毕
```

信封 CSS 设计：
- 主体：圆角矩形，牛皮纸色 `#D4C5B2`，带阴影
- 封口：CSS `clip-path` 三角形，`rotateX` 翻转
- 封蜡：CSS 绘制圆形 + 金色渐变

通知书纸张设计：
- CSS 渐变模拟纸张纹理
- 双线边框（外粗内细）
- 顶部居中"录取通知书"金色渐变文字
- 学生姓名大字号
- 底部"日期：2026年6月"的盖章样式

### 结尾定格

- 背景渐变为略深奶茶色
- "敢赴山海，追梦星辰" 逐字淡入
- "2307，未来可期" 较小字号延迟淡入
- 底部小字："此纪念卡由 2307 班制作"

## 组件结构

```
App
├── HashRouter
│   ├── StudentPage（匹配 /s/:slug）
│   │   ├── PhotoIntro          # 照片轮播
│   │   ├── TeacherMessage      # 教师寄语
│   │   ├── AdmissionLetter     # 通知书展开
│   │   ├── EndingQuote         # 结尾定格
│   │   └── MusicControl        # 音乐控制浮动按钮（预留）
│   └── ErrorPage（slug 未匹配）
```

每个组件职责：
- **PhotoIntro**: 接收 photos 数组，管理轮播生命周期，结束后触发 onDone 回调
- **TeacherMessage**: 接收 quote + author（所有学生收到同一段寄语），管理打字机动画，结束后触发 onDone
- **AdmissionLetter**: 接收 student name + letter config，管理拆信动画，结束后触发 onDone。通知书中只有学生姓名是动态的，其余文案全班统一。
- **EndingQuote**: 接收 className，显示最终定格画面
- **MusicControl**: 浮动按钮，控制背景音乐播放/暂停（MVP 预留接口）
- **ErrorPage**: 独立页面，不依赖学生数据

### 数据流

```
students.json → 静态 import → Map<slug, Student>
config.ts     → 静态 import → siteConfig 对象
                         ↓
              StudentPage 根据 URL slug 查找
                         ↓
              各子组件通过 props 接收数据
```

## MVP 约束

- 图片使用占位图（纯色块或 gradient），后续替换真实照片
- 音乐控制按钮预留，不强制自动播放
- 不做登录、数据库、后台管理
- 不把学生姓名写死在组件里，必须从 students.json 读取
- 保证 `npm install && npm run dev` 可直接运行
- 移动端优先，适配手机竖屏
- 构建产物可直接部署到 Cloudflare Pages

## 照片规格

- 尺寸：1080 × 1920（竖屏全屏）
- 格式：WebP 优先，JPG 备选
- 单张大小上限：≤ 200KB
- 横图使用 `object-fit: cover` 裁切为竖屏全屏，重要人物居中；必要时在 config.ts 中提供 `objectPosition`（focal point）配置，避免合照裁掉边上的同学
- MVP 阶段使用占位图（CSS 渐变），按此规格预留位置

## 图片加载策略

- 第一张照片在 `index.html` 的 `<head>` 中预加载（`<link rel="preload">`）
- 后续照片在轮播播放时预取接下来 2-3 张
- 未加载完成的照片显示占位色块，加载完成后淡入替换
- 所有视口相关尺寸使用 `dvh`（dynamic viewport height）而非 `vh`，适配 iOS Safari 地址栏变化
- 使用 `env(safe-area-inset-*)` 适配刘海屏/挖孔屏

## NFC 标签写入

NFC 标签的 URL 写入不在网页代码层面处理，属于物理部署步骤。使用手机 NFC 工具（如 NFC Tools 等应用）手动将完整 hash URL（如 `https://your-domain.com/#/s/a8k3p2`）写入每个 NFC 贴纸。

## 隐私策略

- 不提供学生列表页，无法通过浏览站点发现其他学生
- slug 使用随机字符串（如 `a8k3p2`），不使用拼音、学号、姓名等可识别信息
- 不使用可枚举编号（如 `/s/001`、`/s/002`），防止遍历
- `index.html` 添加 `<meta name="robots" content="noindex">`
- `public/robots.txt` 禁止搜索引擎抓取
- 页面不展示手机号、真实录取信息、身份证号等敏感信息
- 音乐控制预留但绝不强制自动播放（移动浏览器通常会限制自动播放音频）
